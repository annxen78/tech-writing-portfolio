# UX Text Improvements (ERP System)

## Context

While working on ERP documentation and localization, I identified inconsistencies and unclear wording in the UI.

---

## Problem

- Inconsistent terminology across modules  
- Unclear or unnatural English phrasing  
- Long and confusing UI labels  
- Lack of standard naming conventions  

---

## Approach

I reviewed UI texts and proposed improvements based on:

- clarity and brevity  
- consistency with terminology  
- alignment with user workflows  

---

## Examples

| Module | Current | Improved | Reason |
|--------|--------|---------|--------|
| Preferences | Intercompany sell price | Intercompany Sales Price | Correct wording and more natural phrasing |
| Preferences | multi currency payments | Multicurrency Payments | Standardized capitalization and spelling |
| Preferences | Allow expense even if there are not enough funds in the cashbox | Allow expenses with insufficient funds | Shorter and clearer |
| Finance | Currency exchanges | Currency Conversion | More precise terminology |
| Finance | Money Transfers | Internal Transfers | More accurate |
| Finance | Payments made | Outgoing Payments | Clearer for users |
| Finance | Payments received | Incoming Payments | Clearer for users |

---

## Impact

- Improved clarity of UI texts  
- Reduced ambiguity in financial terminology  
- Increased consistency across modules  
- Better user understanding of system actions  

---


## Additional UX Observations

### Organization Selector

**Problem:**  
The organization selector is not clearly visible and lacks discoverability.

**Suggestion:**  
Increase visibility or reposition it.

---

### Dashboard Layout

**Problem:**  
Key financial indicators are placed in low-visibility areas.

**Suggestion:**  
Move them to the top section of the dashboard.

---

### Icon Consistency

**Problem:**  
Same icons are used for different entities (debtors vs creditors), which creates confusion.

**Suggestion:**  
Use distinct icons or align them with meaning (people vs financial symbols).

---

### Empty State

**Problem:**  
"Empty state" provides no guidance to the user.

**Suggestion:**  
Add actionable hints (e.g. “Add item”).
